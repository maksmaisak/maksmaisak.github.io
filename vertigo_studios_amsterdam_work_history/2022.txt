Change: 775507
Date:   2022-11-18 13:30:15
Description:
[IMP-13686] [IMP-14240]
Fixed Lurker and Nosalis staggering forever in Quest builds. Confirmed working in a Development build.

Did so by fixing Rotator Blackboard Keys not being reset by ClearValue task on Quest.

When the HTNComponent applies worldstate changes made by a task during planning to the blackboard (e.g., the clearing of a StaggerDirection key by a ClearValue task after playing the stagger animation), it first checks if the blackboard already has the same value at that key. If it's the same, it doesn't set the value on blackboard. In a Quest build, the blackboard contains a valid rotator in the StaggerDirection key, and the ClearValue task has planned to replace it with FRotator::InvalidValue (which is FRotator(FLT_MAX)). For some reason, on Quest and only on Quest, the FRotator::Equals comparison returns true for the two values, so it doesn't bother overwriting the old value with the new value, so it stays the same and continues to trigger the stagger behavior forever.

I could not determine why `FRotator(-5.170944, 114.298644, 0.000000).Equals(FAISystem::InvalidRotation)` is true on Quest, so I just removed the comparison: when copying a modified value from worldstate to blackboard, it no longer checks if the values are the same and executes the setting anyway. I don't believe that will have any negative consequences, though the undelying issue that this change dodges (broken FRotator comparison) is worth keeping on our radar.

Vetted by Paul van Zelst

#assetvalidation succeeded
#jira imp-13686,imp-14240
#vettedby pvzelst
